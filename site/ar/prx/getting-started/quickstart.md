# البدء السريع

يأخذك هذا الدليل من الصفر إلى وكيل PRX عامل في أقل من 5 دقائق.

## الخطوة 1: تثبيت PRX

ثبّت أحدث إصدار:

```bash
curl -fsSL https://openprx.dev/install.sh | bash
```

تحقق من التثبيت:

```bash
prx --version
```

::: tip
راجع [دليل التثبيت](./installation) لطرق بديلة (Cargo، البناء من المصدر، Docker).
:::

## الخطوة 2: تشغيل معالج الإعداد الأولي

يضبط معالج الإعداد الأولي مزود النماذج اللغوية ومفتاح API والإعدادات الأولية بشكل تفاعلي:

```bash
prx onboard
```

يرشدك المعالج عبر:

1. **اختيار مزود** -- Anthropic، OpenAI، Ollama، OpenRouter، والمزيد
2. **إدخال مفتاح API** -- يُخزَّن بأمان في ملف الإعدادات
3. **اختيار نموذج افتراضي** -- يجلب المعالج النماذج المتاحة من مزودك
4. **تعيين واجهة الذاكرة** -- Markdown (افتراضي)، SQLite، أو PostgreSQL

بعد اكتمال المعالج، تُحفظ إعداداتك في `~/.config/openprx/openprx.toml`.

::: info إعداد سريع
إذا كنت تعرف مزودك ونموذجك مسبقًا، تجاوز المعالج التفاعلي:

```bash
prx onboard --provider anthropic --api-key sk-ant-... --model claude-sonnet-4-20250514
```

راجع [معالج الإعداد الأولي](./onboarding) لجميع الخيارات.
:::

## الخطوة 3: تشغيل الخادم

شغّل خادم PRX في الخلفية. يدير الخادم بيئة تشغيل الوكيل وواجهة البوابة البرمجية وجميع القنوات المضبوطة:

```bash
prx daemon
```

افتراضيًا، يستمع الخادم على `127.0.0.1:3120`. يمكنك تخصيص المضيف والمنفذ:

```bash
prx daemon --host 0.0.0.0 --port 8080
```

::: tip التشغيل كخدمة
لنشرات الإنتاج، ثبّت PRX كخدمة نظام ليبدأ تلقائيًا عند الإقلاع:

```bash
prx service install
```

ينشئ هذا وحدة systemd (Linux) أو ملف launchd plist (macOS). راجع [prx service](../cli/service) للتفاصيل.
:::

## الخطوة 4: محادثة مع PRX

افتح جلسة محادثة تفاعلية مباشرة في الطرفية:

```bash
prx chat
```

يتصل هذا بالخادم العامل ويفتح واجهة REPL حيث يمكنك التحدث مع النموذج اللغوي المضبوط. اكتب رسالتك واضغط Enter:

```
You: What can you help me with?
PRX: I can help you with a wide range of tasks...
```

يمكنك أيضًا تحديد مزود ونموذج لجلسة واحدة:

```bash
prx chat --provider ollama --model llama3.2
```

اضغط `Ctrl+C` أو اكتب `/quit` للخروج من المحادثة.

## الخطوة 5: ربط قناة

يدعم PRX 19 قناة مراسلة. لربط واحدة، أضف إعداداتها إلى ملف `~/.config/openprx/openprx.toml`.

على سبيل المثال، لربط بوت Telegram:

```toml
[channels.telegram]
bot_token = "123456:ABC-DEF..."
allowed_users = ["your_telegram_username"]
```

ثم أعد تشغيل الخادم لتطبيق القناة الجديدة:

```bash
prx daemon
```

أو استخدم أمر إدارة القنوات:

```bash
prx channel add telegram
```

راجع [نظرة عامة على القنوات](../channels/) للقائمة الكاملة للمنصات المدعومة وإعداداتها.

## الخطوة 6: التحقق من الحالة

اعرض الحالة الحالية لنسخة PRX الخاصة بك:

```bash
prx status
```

يعرض هذا:

- **الإصدار** ومسار الملف التنفيذي
- **مساحة العمل** (المجلد)
- **ملف الإعدادات** (الموقع)
- **المزود** والنموذج المستخدم
- **القنوات النشطة** وحالة اتصالها
- **واجهة الذاكرة** والإحصائيات
- **وقت التشغيل** واستخدام الموارد

مثال على المخرج:

```
PRX Status

Version:     0.3.0
Workspace:   /home/user/.local/share/openprx
Config:      /home/user/.config/openprx/openprx.toml
Provider:    anthropic (claude-sonnet-4-20250514)
Memory:      markdown (/home/user/.local/share/openprx/memory)
Channels:    telegram (connected), cli (active)
Gateway:     http://127.0.0.1:3120
Uptime:      2h 15m
```
