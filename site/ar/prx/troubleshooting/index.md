# استكشاف الأخطاء وإصلاحها

يغطي هذا القسم المشكلات الشائعة التي قد تواجهها أثناء تشغيل PRX وكيفية حلّها.

## تشخيص سريع

شغّل أمر الفحص المدمج لإجراء فحص صحة شامل:

```bash
prx doctor
```

يتحقق هذا من:

- صحة ملف الإعدادات
- اتصال المزوّد والمصادقة
- تبعيات النظام
- مساحة القرص والصلاحيات
- حالة الـ daemon النشطة

## مشكلات شائعة

### الـ Daemon لا يبدأ

**الأعراض**: `prx daemon` يخرج مباشرة أو يفشل في الربط.

**الحلول**:
- تحقق مما إذا كانت هناك نسخة أخرى قيد التشغيل: `prx daemon status`
- تأكد أن المنفذ متاح: `ss -tlnp | grep 3120`
- راجع السجلات: `prx daemon logs`
- تحقق من صحة الإعدادات: `prx config check`

### فشل مصادقة المزوّد

**الأعراض**: أخطاء مثل "Unauthorized" أو "Invalid API key".

**الحلول**:
- تحقق من مفتاح API: `prx auth status`
- أعد تسجيل الدخول: `prx auth login <provider>`
- افحص متغيرات البيئة: `env | grep API_KEY`

### استخدام ذاكرة مرتفع

**الأعراض**: عملية PRX تستهلك ذاكرة بشكل مفرط.

**الحلول**:
- قلّل الجلسات المتزامنة: اضبط `[agent.limits] max_concurrent_sessions`
- فعّل صيانة الذاكرة: `prx memory compact`
- تحقّق من الجلسات طويلة التشغيل: `prx session list`

### تعليق تنفيذ الأدوات

**الأعراض**: يبدو أن الوكيل عالق أثناء تنفيذ الأداة.

**الحلول**:
- تحقّق من إعدادات sandbox
- تأكد من تثبيت تبعيات الأدوات
- اضبط مهلة زمنية: `[agent] session_timeout_secs = 300`
- ألغِ الجلسة: `prx session cancel <id>`
