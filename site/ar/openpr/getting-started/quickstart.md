# البدء السريع

يرشدك هذا الدليل خلال إعداد OpenPR وإنشاء أول مساحة عمل ومشروع ومهام. يفترض أنك أكملت [التثبيت](./installation).

## الخطوة الأولى: بدء OpenPR

إذا لم تبدأ بعد، شغِّل الخدمات:

```bash
cd openpr
docker-compose up -d
```

انتظر حتى تكون جميع الخدمات سليمة:

```bash
docker-compose ps
```

## الخطوة الثانية: تسجيل حساب المسؤول

افتح http://localhost:3000 في متصفحك. انقر **Register** وأنشئ حسابك.

::: tip أول مستخدم هو مسؤول
أول مستخدم يسجل يتلقى دور **admin** تلقائياً. يمكن لهذا المستخدم إدارة جميع مساحات العمل والمشاريع وإعدادات النظام.
:::

## الخطوة الثالثة: إنشاء مساحة عمل

بعد تسجيل الدخول، أنشئ أول مساحة عمل:

1. انقر **Create Workspace** على لوحة التحكم.
2. أدخل اسماً (مثل "My Team") وسبيكة (مثل "my-team").
3. انقر **Create**.

مساحة العمل هي الحاوية العليا لجميع مشاريعك وأعضائك.

## الخطوة الرابعة: إنشاء مشروع

داخل مساحة عملك:

1. انقر **New Project**.
2. أدخل اسماً (مثل "Backend API") ومفتاح مشروع (مثل "API"). يُستخدم المفتاح كبادئة لمعرفات المهام (مثل API-1، API-2).
3. انقر **Create**.

## الخطوة الخامسة: إنشاء مهام

انتقل إلى مشروعك وأنشئ مهام:

1. انقر **New Issue**.
2. أدخل عنواناً ووصفاً.
3. اضبط الحالة (backlog، todo، in_progress، أو done).
4. اختياريًا اضبط الأولوية (low، medium، high، urgent) والمُكلَّف والوسوم.
5. انقر **Create**.

يمكن أيضاً إنشاء المهام عبر API أو خادم MCP:

```bash
# Create an issue via REST API
curl -X POST http://localhost:8080/api/projects/<project_id>/issues \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer <your_token>" \
  -d '{
    "title": "Set up CI pipeline",
    "state": "todo",
    "priority": "high"
  }'
```

## الخطوة السادسة: إعداد لوحة الكانبان

انتقل إلى عرض **Board** في مشروعك. المهام منظمة في أعمدة حسب الحالة:

| العمود | الحالة | الوصف |
|-------|-------|-------|
| Backlog | `backlog` | أفكار وعمل مستقبلي |
| To Do | `todo` | مخطط للدورة الحالية |
| In Progress | `in_progress` | يُعمَل عليه بفاعلية |
| Done | `done` | عمل مكتمل |

اسحب وأسقط المهام بين الأعمدة لتحديث حالتها.

## الخطوة السابعة: دعوة أعضاء الفريق

انتقل إلى **Workspace Settings** > **Members**:

1. انقر **Invite Member**.
2. أدخل عنوان البريد الإلكتروني.
3. اختر دوراً: **Owner** أو **Admin** أو **Member**.

| الدور | الصلاحيات |
|-------|-----------|
| Owner | وصول كامل، يمكن حذف مساحة العمل |
| Admin | إدارة المشاريع والأعضاء والإعدادات |
| Member | إنشاء وإدارة المهام والتعليقات |

## الخطوة الثامنة: ربط مساعدي الذكاء الاصطناعي (اختياري)

أعدّ خادم MCP لتمكين مساعدي الذكاء الاصطناعي من إدارة مشاريعك:

1. انتقل إلى **Workspace Settings** > **Bot Tokens**.
2. أنشئ رمز بوت جديد. سيحمل بادئة `opr_`.
3. هيِّئ مساعد الذكاء الاصطناعي بالرمز.

مثال إعداد Claude Desktop:

```json
{
  "mcpServers": {
    "openpr": {
      "command": "/path/to/mcp-server",
      "args": ["--transport", "stdio"],
      "env": {
        "OPENPR_API_URL": "http://localhost:3000",
        "OPENPR_BOT_TOKEN": "opr_your_token_here",
        "OPENPR_WORKSPACE_ID": "your-workspace-uuid"
      }
    }
  }
}
```

يمكن لمساعد الذكاء الاصطناعي الآن سرد المشاريع وإنشاء المهام وإدارة السبرينت والمزيد من خلال 34 أداة MCP.
