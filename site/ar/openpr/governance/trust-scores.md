# درجات الثقة

درجات الثقة مقياس سمعة لكل مستخدم في OpenPR يتتبع جودة المشاركة وتاريخ اتخاذ القرارات. عند تفعيل التصويت الموزون، تؤثر درجات الثقة مباشرةً على قوة التصويت.

## كيف تعمل درجات الثقة

لكل عضو في مساحة العمل درجة ثقة تعكس مشاركته في الحوكمة:

| العامل | التأثير | الوصف |
|--------|--------|-------|
| جودة المقترح | إيجابي | المقترحات التي تُوافَق عليها ترفع الدرجة |
| المشاركة في التصويت | إيجابي | التصويت المنتظم يرفع الدرجة |
| القرارات المتوافقة | إيجابي | التصويت مع الأغلبية النهائية |
| المقترحات المرفوضة | سلبي | المقترحات التي تُرفض تخفض الدرجة |
| القرارات المنقوضة | سلبي | وجود مقترحاتها منقوضة |
| الاستئنافات | متغير | الاستئنافات الناجحة تستعيد الدرجة |

## نطاق الدرجات

درجات الثقة مُطبَّعة إلى نطاق رقمي. الدرجات الأعلى تشير إلى مشاركة حوكمة أكثر موثوقية:

| النطاق | المستوى | وزن التصويت |
|--------|---------|------------|
| 80-100 | ثقة عالية | وزن 1.5× |
| 50-79 | عادي | وزن 1.0× |
| 20-49 | ثقة منخفضة | وزن 0.75× |
| 0-19 | ضئيل | وزن 0.5× |

::: tip التصويت الموزون
يُطبَّق وزن درجة الثقة فقط عند تفعيل **التصويت الموزون** في إعداد حوكمة مساحة العمل. وإلا، تكون لجميع الأصوات أوزان متساوية.
:::

## عرض درجات الثقة

### عبر واجهة الويب

انتقل إلى **Workspace Settings** > **Governance** > **Trust Scores** لرؤية جميع درجات الأعضاء وسجلاتهم.

### عبر API

```bash
# Get trust scores for the workspace
curl -H "Authorization: Bearer <token>" \
  http://localhost:8080/api/trust-scores

# Get a specific user's trust score history
curl -H "Authorization: Bearer <token>" \
  http://localhost:8080/api/trust-scores/<user_id>/history
```

## سجل درجات الثقة

كل تغيير في درجة الثقة يُسجَّل في جدول `trust_score_logs` مع:

- المستخدم المتأثر
- قيم الدرجة السابقة والجديدة
- سبب التغيير
- الطابع الزمني
- المقترح أو القرار المرتبط (إن وجد)

يوفر هذا السجل شفافية في كيفية تطور الدرجات عبر الزمن.

## الاستئنافات

إذا اعتقد عضو أن درجة ثقته تأثرت بشكل غير عادل، يمكنه تقديم استئناف:

1. انتقل إلى سجل درجة الثقة الخاص به.
2. انقر **Appeal** على تغيير درجة معينة.
3. قدِّم سبباً للاستئناف.
4. يراجع مسؤولو مساحة العمل ويبتون في الاستئناف.

الاستئنافات الناجحة تستعيد تغيير الدرجة. تُحفظ سجلات الاستئناف في سجل التدقيق.

```bash
# File an appeal
curl -X POST http://localhost:8080/api/trust-scores/<user_id>/appeals \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer <token>" \
  -d '{"reason": "Score decreased due to a test proposal that was not meant for production."}'
```

## مراجعات التأثير

درجات الثقة أحد المدخلات في **مراجعات التأثير** -- تقييمات لكيفية تأثير مقترح أو قرار على المشروع. مراجعات التأثير تتضمن:

- المقاييس الكمية (الجهد التقديري، مستوى الخطر، النطاق)
- التقييمات النوعية من المشاركين في المراجعة
- البيانات التاريخية من القرارات المماثلة
