# L3: Эволюция стратегий

Уровень 3 -- это наиболее влиятельный и наименее частый уровень эволюции. Он модифицирует стратегическое поведение агента -- политики доступа к инструментам, правила маршрутизации LLM и параметры управления. Из-за широкого воздействия изменения L3 всегда требуют явного одобрения.

## Обзор

L3-эволюция охватывает:

- **Настройку политик инструментов** -- корректировку доступных инструментов и их уровней разрешений
- **Оптимизацию маршрутизации** -- уточнение эвристик выбора модели на основе данных производительности
- **Параметры управления** -- настройку порогов безопасности, ограничений частоты и процессов одобрения
- **Оптимизацию затрат** -- баланс качества и использования ресурсов

## Как это работает

1. L3 собирает агрегированные данные производительности за недели/месяцы
2. Определяет закономерности (напр., более дешёвая модель адекватно обрабатывает 80% запросов)
3. Предлагает стратегические изменения с анализом ожидаемого воздействия
4. Изменения ставятся в очередь на одобрение человеком
5. Одобренные изменения применяются с возможностью автоматического отката

## Конфигурация

```toml
[self_evolution.l3]
enabled = false
schedule = "monthly"
require_approval = true
rollback_window_hours = 168  # 7 дней
max_policy_changes_per_cycle = 3
```
