# Быстрый старт

Это руководство позволит вам перейти от нуля к работающему агенту PRX менее чем за 5 минут.

## Шаг 1: Установка PRX

Установите последний релиз:

```bash
curl -fsSL https://openprx.dev/install.sh | bash
```

Проверьте установку:

```bash
prx --version
```

::: tip
Альтернативные способы установки (Cargo, сборка из исходного кода, Docker) описаны в [Руководстве по установке](./installation).
:::

## Шаг 2: Запуск мастера настройки

Мастер настройки в интерактивном режиме конфигурирует ваш LLM-провайдер, API-ключ и начальные параметры:

```bash
prx onboard
```

Мастер проведёт вас через следующие этапы:

1. **Выбор провайдера** — Anthropic, OpenAI, Ollama, OpenRouter и другие
2. **Ввод API-ключа** — ключ безопасно сохраняется в файле конфигурации
3. **Выбор модели по умолчанию** — мастер запрашивает список доступных моделей у вашего провайдера
4. **Выбор бэкенда памяти** — Markdown (по умолчанию), SQLite или PostgreSQL

После завершения работы мастера конфигурация сохраняется в `~/.config/openprx/openprx.toml`.

::: info Быстрая настройка
Если вы уже знаете своего провайдера и модель, можно пропустить интерактивный мастер:

```bash
prx onboard --provider anthropic --api-key sk-ant-... --model claude-sonnet-4-20250514
```

Все доступные параметры описаны в разделе [Мастер настройки](./onboarding).
:::

## Шаг 3: Запуск демона

Запустите демон PRX в фоновом режиме. Демон управляет средой выполнения агента, шлюзовым API и всеми настроенными каналами:

```bash
prx daemon
```

По умолчанию демон слушает на `127.0.0.1:3120`. Хост и порт можно настроить:

```bash
prx daemon --host 0.0.0.0 --port 8080
```

::: tip Запуск как сервис
Для промышленных развёртываний установите PRX как системный сервис, чтобы он запускался автоматически при загрузке:

```bash
prx service install
```

Это создаёт юнит systemd (Linux) или plist launchd (macOS). Подробнее — в разделе [prx service](../cli/service).
:::

## Шаг 4: Чат с PRX

Откройте интерактивную сессию чата прямо в терминале:

```bash
prx chat
```

Это подключается к работающему демону и открывает REPL, в котором вы можете общаться с настроенной LLM. Введите сообщение и нажмите Enter:

```
You: What can you help me with?
PRX: I can help you with a wide range of tasks...
```

Вы также можете указать провайдера и модель для отдельной сессии:

```bash
prx chat --provider ollama --model llama3.2
```

Нажмите `Ctrl+C` или введите `/quit`, чтобы выйти из чата.

## Шаг 5: Подключение канала

PRX поддерживает 19 каналов обмена сообщениями. Чтобы подключить канал, добавьте его конфигурацию в файл `~/.config/openprx/openprx.toml`.

Например, для подключения Telegram-бота:

```toml
[channels.telegram]
bot_token = "123456:ABC-DEF..."
allowed_users = ["your_telegram_username"]
```

Затем перезапустите демон для применения нового канала:

```bash
prx daemon
```

Или используйте команду управления каналами:

```bash
prx channel add telegram
```

Полный список поддерживаемых платформ и их конфигурация описаны в [Обзоре каналов](../channels/).

## Шаг 6: Проверка статуса

Просмотр текущего состояния вашего экземпляра PRX:

```bash
prx status
```

Отображаемая информация:

- **Версия** и путь к бинарнику
- **Рабочая область** — каталог данных
- **Конфигурация** — расположение файла конфигурации
- **Провайдер** и используемая модель
- **Активные каналы** и их статус подключения
- **Бэкенд памяти** и статистика
- **Время работы** и использование ресурсов

Пример вывода:

```
PRX Status

Version:     0.3.0
Workspace:   /home/user/.local/share/openprx
Config:      /home/user/.config/openprx/openprx.toml
Provider:    anthropic (claude-sonnet-4-20250514)
Memory:      markdown (/home/user/.local/share/openprx/memory)
Channels:    telegram (connected), cli (active)
Gateway:     http://127.0.0.1:3120
Uptime:      2h 15m
```
