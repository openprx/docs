# Управление спринтами

Спринты — ограниченные по времени итерации для организации и отслеживания работы. Каждый спринт принадлежит проекту и имеет дату начала, дату окончания и набор назначенных задач.

## Создание спринта

### Через веб-интерфейс

1. Перейдите в ваш проект.
2. Перейдите в раздел **Sprints**.
3. Нажмите **New Sprint**.
4. Введите название спринта, дату начала и дату окончания.

### Через API

```bash
curl -X POST http://localhost:8080/api/projects/<project_id>/sprints \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer <token>" \
  -d '{
    "name": "Sprint 1",
    "start_date": "2026-03-24",
    "end_date": "2026-04-07"
  }'
```

### Через MCP

```json
{
  "method": "tools/call",
  "params": {
    "name": "sprints.create",
    "arguments": {
      "project_id": "<project_uuid>",
      "name": "Sprint 1",
      "start_date": "2026-03-24",
      "end_date": "2026-04-07"
    }
  }
}
```

## Поля спринта

| Поле | Тип | Обязательное | Описание |
|------|-----|-------------|----------|
| Название | string | Да | Название спринта (например, "Sprint 1", "Q1 Week 3") |
| Дата начала | date | Нет | Дата начала спринта |
| Дата окончания | date | Нет | Дата окончания спринта |
| Статус | enum | Авто | Активный, завершённый или запланированный |

## Назначение задач спринтам

Назначьте задачи спринту, обновив `sprint_id` задачи:

```bash
curl -X PATCH http://localhost:8080/api/issues/<issue_id> \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer <token>" \
  -d '{"sprint_id": "<sprint_uuid>"}'
```

Или через веб-интерфейс перетащите задачи в раздел спринта или воспользуйтесь панелью деталей задачи.

## Рабочий процесс планирования спринта

Типичный рабочий процесс планирования спринта:

1. **Создайте спринт** с датами начала и окончания.
2. **Просмотрите бэклог** — определите задачи для включения.
3. **Переместите задачи** из Backlog/To Do в спринт.
4. **Установите приоритеты** и исполнителей для задач спринта.
5. **Запустите спринт** — команда приступает к работе.
6. **Отслеживайте прогресс** на доске и в виде спринта.
7. **Завершите спринт** — проверьте выполненные/оставшиеся элементы.

## MCP-инструменты

| Инструмент | Параметры | Описание |
|----------|---------|----------|
| `sprints.list` | `project_id` | Список спринтов в проекте |
| `sprints.create` | `project_id`, `name` | Создать спринт с опциональными датами |
| `sprints.update` | `sprint_id` | Обновить название, даты или статус |
| `sprints.delete` | `sprint_id` | Удалить спринт |
