# პროვაიდერის პროფილები

პროვაიდერის პროფილები საშუალებას გაძლევთ ერთი და იმავე პროვაიდერისთვის მრავალი ავთენტიფიკაციის კონტექსტის კონფიგურაცია. ეს სასარგებლოა, როდესაც გაქვთ ცალკე ანგარიშები პირადი და სამუშაო გამოყენებისთვის, ან როდესაც დეველოპმენტისა და პროდაქშენის API გასაღებებს შორის გადართავთ.

## მიმოხილვა

პროფილი არის სახელდებული კონფიგურაცია, რომელიც მოიცავს:

- პროვაიდერის იდენტიფიკატორს
- ავთენტიფიკაციის სანდოობის მონაცემებს (API გასაღები ან OAuth2 ტოკენები)
- მოდელის პრეფერენციებს
- სიჩქარის ლიმიტის გადაფარვებს

## კონფიგურაცია

```toml
[[auth.profiles]]
name = "personal"
provider = "anthropic"
api_key = "sk-ant-personal-..."
default_model = "claude-haiku"

[[auth.profiles]]
name = "work"
provider = "anthropic"
api_key = "sk-ant-work-..."
default_model = "claude-sonnet-4-6"
```

## პროფილების გადართვა

```bash
# კონკრეტული პროფილის გამოყენება
prx chat --profile work

# ნაგულისხმევი პროფილის დაყენება
prx auth set-default work

# პროფილების სიის ნახვა
prx auth profiles
```

## გარემოს ცვლადები

პროფილებს შეუძლიათ გარემოს ცვლადებზე მითითება სანდოობის მონაცემებისთვის:

```toml
[[auth.profiles]]
name = "ci"
provider = "anthropic"
api_key = "${ANTHROPIC_API_KEY}"
```
