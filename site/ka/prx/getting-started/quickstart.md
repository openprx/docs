# სწრაფი დაწყება

ეს სახელმძღვანელო გამოშვებულ PRX აგენტამდე მიგიყვანთ 5 წუთზე ნაკლებ დროში.

## ნაბიჯი 1: დააინსტალირეთ PRX

დააინსტალირეთ უახლესი რელიზი:

```bash
curl -fsSL https://openprx.dev/install.sh | bash
```

შეამოწმეთ ინსტალაცია:

```bash
prx --version
```

::: tip
იხილეთ [ინსტალაციის სახელმძღვანელო](./installation) ალტერნატიული მეთოდებისთვის (Cargo, წყაროდან აგება, Docker).
:::

## ნაბიჯი 2: გაუშვით ონბორდინგის ოსტატი

ონბორდინგის ოსტატი ინტერაქტიულად აკონფიგურირებს თქვენს LLM პროვაიდერს, API გასაღებს და საწყის პარამეტრებს:

```bash
prx onboard
```

ოსტატი გაგატარებთ შემდეგ ეტაპებზე:

1. **პროვაიდერის არჩევა** -- Anthropic, OpenAI, Ollama, OpenRouter და სხვა
2. **API გასაღების შეყვანა** -- უსაფრთხოდ ინახება კონფიგურაციის ფაილში
3. **ნაგულისხმევი მოდელის არჩევა** -- ოსტატი იღებს ხელმისაწვდომ მოდელების სიას თქვენი პროვაიდერიდან
4. **მეხსიერების ბექენდის დაყენება** -- Markdown (ნაგულისხმევი), SQLite ან PostgreSQL

ოსტატის დასრულების შემდეგ თქვენი კონფიგურაცია შენახულია `~/.config/openprx/openprx.toml`-ში.

::: info სწრაფი დაყენება
თუ უკვე იცით თქვენი პროვაიდერი და მოდელი, გამოტოვეთ ინტერაქტიული ოსტატი:

```bash
prx onboard --provider anthropic --api-key sk-ant-... --model claude-sonnet-4-20250514
```

იხილეთ [ონბორდინგის ოსტატი](./onboarding) ყველა ვარიანტისთვის.
:::

## ნაბიჯი 3: გაუშვით დემონი

გაუშვით PRX დემონი ფონურ რეჟიმში. დემონი მართავს აგენტის გაშვების გარემოს, გეითვეის API-ს და ყველა დაკონფიგურირებულ არხს:

```bash
prx daemon
```

ნაგულისხმევად, დემონი უსმენს `127.0.0.1:3120`-ზე. შეგიძლიათ მოარგოთ ჰოსტი და პორტი:

```bash
prx daemon --host 0.0.0.0 --port 8080
```

::: tip სერვისის სახით გაშვება
წარმოების განთავსებებისთვის დააინსტალირეთ PRX სისტემურ სერვისად, რომ ავტომატურად დაიწყოს ჩატვირთვისას:

```bash
prx service install
```

ეს ქმნის systemd ერთეულს (Linux) ან launchd plist-ს (macOS). იხილეთ [prx service](../cli/service) დეტალებისთვის.
:::

## ნაბიჯი 4: ესაუბრეთ PRX-ს

გახსენით ინტერაქტიული ჩატის სესია პირდაპირ თქვენს ტერმინალში:

```bash
prx chat
```

ეს უკავშირდება გაშვებულ დემონს და ხსნის REPL-ს, სადაც შეგიძლიათ ესაუბროთ თქვენს დაკონფიგურირებულ LLM-ს. ჩაწერეთ თქვენი შეტყობინება და დააჭირეთ Enter:

```
You: What can you help me with?
PRX: I can help you with a wide range of tasks...
```

ასევე შეგიძლიათ მიუთითოთ პროვაიდერი და მოდელი ერთი სესიისთვის:

```bash
prx chat --provider ollama --model llama3.2
```

დააჭირეთ `Ctrl+C` ან ჩაწერეთ `/quit` ჩატიდან გასასვლელად.

## ნაბიჯი 5: დააკავშირეთ არხი

PRX მხარს უჭერს 19 შეტყობინებების არხს. დასაკავშირებლად, დაამატეთ მისი კონფიგურაცია თქვენს `~/.config/openprx/openprx.toml` ფაილში.

მაგალითად, Telegram ბოტის დასაკავშირებლად:

```toml
[channels.telegram]
bot_token = "123456:ABC-DEF..."
allowed_users = ["your_telegram_username"]
```

შემდეგ გადატვირთეთ დემონი ახალი არხის ასაღებად:

```bash
prx daemon
```

ან გამოიყენეთ არხის მართვის ბრძანება:

```bash
prx channel add telegram
```

იხილეთ [არხების მიმოხილვა](../channels/) მხარდაჭერილი პლატფორმების სრული სიისა და მათი კონფიგურაციისთვის.

## ნაბიჯი 6: შეამოწმეთ სტატუსი

ნახეთ თქვენი PRX ინსტანციის ამჟამინდელი მდგომარეობა:

```bash
prx status
```

ეს აჩვენებს:

- **ვერსიას** და ბინარის გზას
- **სამუშაო სივრცის** დირექტორიას
- **კონფიგურაციის** ფაილის მდებარეობას
- გამოყენებულ **პროვაიდერს** და მოდელს
- **აქტიურ არხებს** და მათ კავშირის სტატუსს
- **მეხსიერების ბექენდს** და სტატისტიკას
- **მუშაობის დროს** და რესურსების გამოყენებას

გამოსავლის მაგალითი:

```
PRX Status

Version:     0.3.0
Workspace:   /home/user/.local/share/openprx
Config:      /home/user/.config/openprx/openprx.toml
Provider:    anthropic (claude-sonnet-4-20250514)
Memory:      markdown (/home/user/.local/share/openprx/memory)
Channels:    telegram (connected), cli (active)
Gateway:     http://127.0.0.1:3120
Uptime:      2h 15m
```
