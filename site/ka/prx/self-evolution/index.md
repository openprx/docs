# თვითევოლუციის სისტემა

PRX მოიცავს 3-ფენიან თვითევოლუციის სისტემას, რომელიც აგენტებს დროთა განმავლობაში ქცევის ავტონომიურად გაუმჯობესების საშუალებას აძლევს. სისტემა უწყვეტად აანალიზებს აგენტის წარმადობას და იყენებს ეტაპობრივ გაუმჯობესებებს -- მეხსიერების ოპტიმიზაციიდან პრომფთის დახვეწამდე და სტრატეგიული პოლიტიკის ცვლილებებამდე.

## მიმოხილვა

თვითევოლუცია სამ ფენად არის ორგანიზებული, თითოეული აბსტრაქციის სხვადასხვა დონეზე მოქმედებს:

| ფენა | სფერო | სიხშირე | რისკი |
|------|-------|---------|-------|
| [L1: მეხსიერება](./l1-memory) | მეხსიერების შეკუმშვა, თემატური კლასტერიზაცია | ყოველი სესია | დაბალი |
| [L2: პრომფთი](./l2-prompt) | სისტემური პრომფთის ოპტიმიზაცია, A/B ტესტირება | ყოველდღიური/ყოველკვირეული | საშუალო |
| [L3: სტრატეგია](./l3-strategy) | ინსტრუმენტების პოლიტიკები, მარშრუტიზაციის წესები, მმართველობა | ყოველკვირეული/ყოველთვიური | მაღალი |

## უსაფრთხოება პირველ ადგილზე

ყოველი ევოლუციის წინადადება უსაფრთხოების პაიპლაინს გადის შესრულებამდე. იხილეთ [უსაფრთხოება](./safety) უკანდაბრუნების დაცვისა და გონიერების შემოწმებების დეტალებისთვის.

## კონფიგურაცია

```toml
[self_evolution]
enabled = false  # მხოლოდ ხელით ჩართვით
auto_apply = false  # ნაგულისხმევად მოითხოვს ხელით დამტკიცებას

[self_evolution.l1]
enabled = true
schedule = "after_session"

[self_evolution.l2]
enabled = false
schedule = "weekly"

[self_evolution.l3]
enabled = false
schedule = "monthly"
require_approval = true
```
