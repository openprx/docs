# CLI მითითება

`prx` ბინარი არის ერთიანი შესასვლელი წერტილი ყველა PRX ოპერაციისთვის -- ინტერაქტიული ჩატი, დემონის მართვა, არხების ადმინისტრირება და სისტემის დიაგნოსტიკა.

## გლობალური ფლაგები

ეს ფლაგები მიღებულია ყველა ქვებრძანების მიერ.

| ფლაგი | მოკლე | ნაგულისხმევი | აღწერა |
|-------|-------|-------------|--------|
| `--config` | `-c` | `~/.config/prx/config.toml` | კონფიგურაციის ფაილის გზა |
| `--log-level` | `-l` | `info` | ლოგის სიტყვიერება: `trace`, `debug`, `info`, `warn`, `error` |
| `--no-color` | | `false` | ფერადი გამოსავლის გამორთვა |
| `--quiet` | `-q` | `false` | არაარსებითი გამოსავლის დახშობა |
| `--help` | `-h` | | დახმარების ინფორმაციის ჩვენება |
| `--version` | `-V` | | ვერსიის ჩვენება |

## ბრძანებები

| ბრძანება | აღწერა |
|----------|--------|
| [`prx agent`](./agent) | ერთჯერადი LLM ურთიერთქმედება (მილის თავსებადი) |
| [`prx chat`](./chat) | მდიდარი ტერმინალის ჩატი სტრიმინგითა და ისტორიით |
| [`prx daemon`](./daemon) | სრული PRX გაშვების გარემოს გაშვება (გეითვეი + არხები + cron + ევოლუცია) |
| [`prx gateway`](./gateway) | ცალკეული HTTP/WebSocket გეითვეი სერვერი |
| [`prx onboard`](./onboard) | ინტერაქტიული დაყენების ოსტატი |
| [`prx channel`](./channel) | არხების მართვა (სია, დამატება, წაშლა, გაშვება, ექიმი) |
| [`prx cron`](./cron) | Cron ამოცანების მართვა (სია, დამატება, წაშლა, პაუზა, განახლება) |
| [`prx evolution`](./evolution) | თვითევოლუციის ოპერაციები (სტატუსი, ისტორია, კონფიგურაცია, ტრიგერი) |
| [`prx auth`](./auth) | OAuth პროფილების მართვა (შესვლა, განახლება, გასვლა) |
| [`prx config`](./config) | კონფიგურაციის ოპერაციები (სქემა, გაყოფა, შერწყმა, მიღება, დაყენება) |
| [`prx doctor`](./doctor) | სისტემის დიაგნოსტიკა (დემონის ჯანმრთელობა, არხის სტატუსი, მოდელის ხელმისაწვდომობა) |
| [`prx service`](./service) | Systemd/OpenRC სერვისის მართვა (ინსტალაცია, გაშვება, შეჩერება, სტატუსი) |
| [`prx skills`](./skills) | უნარების მართვა (სია, ინსტალაცია, წაშლა) |
| `prx status` | სისტემის სტატუსის დაფა |
| `prx models refresh` | პროვაიდერის მოდელების კატალოგების განახლება |
| `prx providers` | ყველა მხარდაჭერილი LLM პროვაიდერის ჩამოთვლა |
| `prx completions` | Shell-ის ავტოდასრულებების გენერირება (bash, zsh, fish) |

## სწრაფი მაგალითები

```bash
# პირველი დაყენება
prx onboard

# ინტერაქტიული ჩატის დაწყება
prx chat

# ერთჯერადი მოთხოვნა (სკრიპტებისთვის)
echo "Summarize this file" | prx agent -f report.pdf

# დემონის გაშვება ყველა სერვისით
prx daemon

# სისტემის ჯანმრთელობის შემოწმება
prx doctor
```

## Shell-ის ავტოდასრულებები

შექმენით ავტოდასრულებები თქვენი shell-ისთვის და დაამატეთ თქვენს პროფილში:

```bash
# Bash
prx completions bash > ~/.local/share/bash-completion/completions/prx

# Zsh
prx completions zsh > ~/.zfunc/_prx

# Fish
prx completions fish > ~/.config/fish/completions/prx.fish
```

## გარემოს ცვლადები

PRX პატივს სცემს შემდეგ გარემოს ცვლადებს (ეს მნიშვნელობები უპირატესია კონფიგურაციის ფაილის მნიშვნელობებზე):

| ცვლადი | აღწერა |
|--------|--------|
| `PRX_CONFIG` | კონფიგურაციის ფაილის გზა (იგივეა, რაც `--config`) |
| `PRX_LOG` | ლოგის დონე (იგივეა, რაც `--log-level`) |
| `PRX_DATA_DIR` | მონაცემების დირექტორია (ნაგულისხმევი: `~/.local/share/prx`) |
| `ANTHROPIC_API_KEY` | Anthropic პროვაიდერის API გასაღები |
| `OPENAI_API_KEY` | OpenAI პროვაიდერის API გასაღები |
| `GOOGLE_API_KEY` | Google Gemini პროვაიდერის API გასაღები |
