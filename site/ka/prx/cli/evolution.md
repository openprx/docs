# prx evolution

თვითევოლუციის ძრავის შემოწმება და კონტროლი. PRX მხარს უჭერს ავტონომიური ევოლუციის სამ დონეს: L1 (მეხსიერება), L2 (პრომფთები) და L3 (სტრატეგიები). ეს ბრძანება საშუალებას გაძლევთ შეამოწმოთ ევოლუციის სტატუსი, განიხილოთ ისტორია, განაახლოთ კონფიგურაცია და ხელით გაუშვათ ევოლუციის ციკლები.

## გამოყენება

```bash
prx evolution <SUBCOMMAND> [OPTIONS]
```

## ქვებრძანებები

### `prx evolution status`

ევოლუციის ძრავის მიმდინარე მდგომარეობის ჩვენება.

```bash
prx evolution status [OPTIONS]
```

| ფლაგი | მოკლე | ნაგულისხმევი | აღწერა |
|-------|-------|-------------|--------|
| `--json` | `-j` | `false` | გამოტანა JSON ფორმატში |

**გამოსავლის მაგალითი:**

```
 Evolution Engine Status
 ───────────────────────
 Engine:    running
 L1 Memory:    enabled   (last: 2h ago, next: in 4h)
 L2 Prompt:    enabled   (last: 1d ago, next: in 23h)
 L3 Strategy:  disabled
 Total cycles: 142
 Rollbacks:    3
```

### `prx evolution history`

ევოლუციის ისტორიის ჟურნალის ჩვენება.

```bash
prx evolution history [OPTIONS]
```

| ფლაგი | მოკლე | ნაგულისხმევი | აღწერა |
|-------|-------|-------------|--------|
| `--limit` | `-n` | `20` | საჩვენებელი ჩანაწერების რაოდენობა |
| `--level` | `-l` | ყველა | დონით ფილტრაცია: `l1`, `l2`, `l3` |
| `--json` | `-j` | `false` | გამოტანა JSON ფორმატში |

```bash
# ბოლო 10 L2 ევოლუციის ჩვენება
prx evolution history --limit 10 --level l2
```

**გამოსავლის მაგალითი:**

```
 Time                Level  Action                          Status
 2026-03-21 08:00    L1     memory consolidation            success
 2026-03-20 20:00    L1     memory consolidation            success
 2026-03-20 09:00    L2     prompt refinement (system)      success
 2026-03-19 14:22    L2     prompt refinement (tool-use)    rolled back
```

### `prx evolution config`

ევოლუციის კონფიგურაციის ნახვა ან განახლება.

```bash
prx evolution config [OPTIONS]
```

| ფლაგი | მოკლე | ნაგულისხმევი | აღწერა |
|-------|-------|-------------|--------|
| `--set` | | | კონფიგურაციის მნიშვნელობის დაყენება (მაგ., `--set l1.enabled=true`) |
| `--json` | `-j` | `false` | გამოტანა JSON ფორმატში |

```bash
# მიმდინარე კონფიგურაციის ნახვა
prx evolution config

# L3 სტრატეგიის ევოლუციის ჩართვა
prx evolution config --set l3.enabled=true

# L1 ინტერვალის დაყენება 2 საათზე
prx evolution config --set l1.interval=7200
```

### `prx evolution trigger`

ევოლუციის ციკლის ხელით გაშვება.

```bash
prx evolution trigger [OPTIONS]
```

| ფლაგი | მოკლე | ნაგულისხმევი | აღწერა |
|-------|-------|-------------|--------|
| `--level` | `-l` | `l1` | გასაშვები ევოლუციის დონე: `l1`, `l2`, `l3` |
| `--dry-run` | | `false` | ევოლუციის წინასწარი ნახვა ცვლილებების გამოყენების გარეშე |

```bash
# L1 მეხსიერების ევოლუციის გაშვება
prx evolution trigger --level l1

# L2 პრომფთის ევოლუციის წინასწარი ნახვა
prx evolution trigger --level l2 --dry-run
```

## ევოლუციის დონეები

| დონე | სამიზნე | აღწერა |
|------|---------|--------|
| **L1** | მეხსიერება | მეხსიერების ჩანაწერების კონსოლიდაცია, დედუპლიკაცია და ორგანიზება |
| **L2** | პრომფთები | სისტემური პრომფთებისა და ინსტრუმენტების გამოყენების ინსტრუქციების დახვეწა ურთიერთქმედების შაბლონების საფუძველზე |
| **L3** | სტრატეგიები | მაღალი დონის ქცევითი სტრატეგიების ადაპტაცია (მოითხოვს მკაფიო ჩართვას) |

ყველა ევოლუციის ცვლილება შექცევადია. ძრავა ინახავს დაბრუნების ისტორიას და ავტომატურად აბრუნებს ცვლილებებს, რომლებიც იწვევენ წარმადობის გაუარესებას.
