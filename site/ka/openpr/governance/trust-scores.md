# ნდობ-ქულები

ნდობ-ქულები OpenPR-ში თითო-მომხმარებლ-რეპუტ-მეტრიკაა, რომელიც მონაწილეობ-ხარისხსა და გადაწყვეტილებ-მიღებ-ისტორიას ადევნებს. შეწონილი ხმ-მიცემ-ჩართვისას ნდობ-ქულები ხმ-ძალაზე პირდაპირ გავლენას ახდენს.

## ნდობ-ქულების მუშაობა

ყოველ სამუშაო სივრც-წევრს ნდობ-ქულა აქვს, რომელიც მმართველობ-მონაწილეობას ასახავს:

| ფაქტორი | გავლენა | აღწერა |
|--------|--------|-------------|
| წინადადებ-ხარისხი | დადებითი | დამტკიცებული წინადადებები ქულას ზრდის |
| ხმ-მიცემ-მონაწილეობა | დადებითი | რეგულარული ხმ-მიცემა ქულას ზრდის |
| გასწორებული გადაწყვეტილებები | დადებითი | საბოლოო უმრავლესობასთან ხმ-მიცემა |
| უარყოფილი წინადადებები | უარყოფითი | უარყოფილი წინადადებები ქულას ამცირებს |
| ვეტო-დადებული გადაწყვეტილებები | უარყოფითი | წინადადებებზე ვეტოს დადება |
| გასაჩივრებები | ცვლადი | წარმატებული გასაჩივრებები ქულას აღადგენს |

## ქულ-დიაპაზონი

ნდობ-ქულები რიცხვით დიაპაზონამდეა ნორმირებული. მაღალი ქულები მმართველობ-მონაწილეობის სანდოობას მიუთითებს:

| დიაპაზონი | დონე | ხმ-წონა |
|-------|-------|---------------|
| 80-100 | მაღალი ნდობა | 1.5x წონა |
| 50-79 | ნორმალური | 1.0x წონა |
| 20-49 | დაბალი ნდობა | 0.75x წონა |
| 0-19 | მინიმალური | 0.5x წონა |

::: tip შეწონილი ხმ-მიცემა
ნდობ-ქულ-შეწონვა მხოლოდ სამუშაო სივრც-მმართველობ-კონფ-ში **შეწონილი ხმ-მიცემის** ჩართვისას გამოიყენება. სხვა შემთხვევაში ყველა ხმა თანაბარი წონის მქონეა.
:::

## ნდობ-ქულების ნახვა

### ვებ UI-ის გავლით

**Workspace Settings** > **Governance** > **Trust Scores**-ზე გადასვლა ყველა წევრ-ქულისა და ისტორიის სანახავად.

### API-ის გავლით

```bash
# Get trust scores for the workspace
curl -H "Authorization: Bearer <token>" \
  http://localhost:8080/api/trust-scores

# Get a specific user's trust score history
curl -H "Authorization: Bearer <token>" \
  http://localhost:8080/api/trust-scores/<user_id>/history
```

## ნდობ-ქულ-ისტორია

ნდობ-ქულ-ყოველი ცვლილება `trust_score_logs` ცხრილში ჩაიწერება:

- ზეგავლენილი მომხმარებელი
- წინა და ახალი ქულ-მნიშვნელობები
- ცვლილებ-მიზეზი
- დრო
- დაკავშირებული წინადადება ან გადაწყვეტილება (ასეთის შემთხვევაში)

ეს ისტორია ქულების დროთა განმავლობაში ევოლუციის გამჭვირვალეობას უზრუნველყოფს.

## გასაჩივრებები

წევრს ნდობ-ქულის უსამართლოდ ზეგავლენის შემთხვევაში გასაჩივრება შეუძლია:

1. ნდობ-ქულ-ისტორიაზე გადასვლა.
2. კონკრ-ქულ-ცვლილებაზე **Appeal**-ზე დაჭერა.
3. გასაჩივრებ-მიზეზის მიწოდება.
4. სამუშაო სივრც-ადმინები განხილავენ და გასაჩივრებაზე გადაწყვეტილებას მიიღებენ.

წარმატებული გასაჩივრებები ქულ-ცვლილებას აღადგენს. გასაჩივრებ-ჩანაწერები აუდიტ-ლოგში ინახება.

```bash
# File an appeal
curl -X POST http://localhost:8080/api/trust-scores/<user_id>/appeals \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer <token>" \
  -d '{"reason": "Score decreased due to a test proposal that was not meant for production."}'
```

## გავლენ-მიმოხილვები

ნდობ-ქულები **გავლენ-მიმოხილვების** ერთ-ერთი შეყვანაა -- წინადადების ან გადაწყვეტილების პროექტზე გავლენის შეფასება. გავლენ-მიმოხილვები მოიცავს:

- რაოდენობრივ მეტრიკებს (სავარაუდო ძალისხმევა, რისკ-დონე, სფერო)
- მიმოხილვ-მონაწილეების თვისობრივ შეფასებებს
- ანალოგიური გადაწყვეტილებების ისტორიულ მონაცემებს
