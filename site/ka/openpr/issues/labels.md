# ეტიკეტები

ეტიკეტები issue-ების კატეგორიზება-ფილტრაციის მოქნილ გზას იძლევა. ყოველ ეტიკეტს სახელი, ფერი და სურვილისამებრ აღწერა აქვს.

## ეტიკეტ-შექმნა

### ვებ UI-ის გავლით

1. პროექტ-ან სამუშაო სივრც-პარამეტრებზე გადასვლა.
2. **Labels**-ზე გადასვლა.
3. **New Label**-ზე დაჭერა.
4. სახელის შეყვანა (მაგ., "bug", "feature", "documentation").
5. ფერის შერჩევა (hex-ფორმატი, მაგ., `#ef4444` წითლისთვის).
6. **Create**-ზე დაჭერა.

### API-ის გავლით

```bash
curl -X POST http://localhost:8080/api/labels \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer <token>" \
  -d '{
    "name": "bug",
    "color": "#ef4444",
    "description": "Something is not working"
  }'
```

### MCP-ის გავლით

```json
{
  "method": "tools/call",
  "params": {
    "name": "labels.create",
    "arguments": {
      "name": "bug",
      "color": "#ef4444"
    }
  }
}
```

## Issue-ებზე ეტიკეტ-დამატება

### ვებ UI-ის გავლით

Issue-ს გახსენი და ეტიკეტ-დამატება-ამოღებისთვის **Labels** ველზე დაჭერა.

### API-ის გავლით

```bash
# Add a label to an issue
curl -X POST http://localhost:8080/api/issues/<issue_id>/labels \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer <token>" \
  -d '{"label_id": "<label_uuid>"}'
```

### MCP-ის გავლით

| ინსტრუმენტი | პარამეტრები | აღწერა |
|------|--------|-------------|
| `work_items.add_label` | `work_item_id`, `label_id` | ერთი ეტიკეტის დამატება |
| `work_items.add_labels` | `work_item_id`, `label_ids` | მრავალი ეტიკეტის დამატება |
| `work_items.remove_label` | `work_item_id`, `label_id` | ეტიკეტის ამოღება |
| `work_items.list_labels` | `work_item_id` | issue-ს ეტიკეტების ჩამოთვლა |

## ეტიკეტ-მართვ-MCP ინსტრუმენტები

| ინსტრუმენტი | პარამეტრები | აღწერა |
|------|--------|-------------|
| `labels.list` | -- | ყველა სამუშაო სივრც-ეტიკეტის ჩამოთვლა |
