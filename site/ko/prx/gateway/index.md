# 게이트웨이

PRX 게이트웨이는 여러 프로토콜을 통해 에이전트 기능을 노출하는 네트워크 대면 계층입니다. HTTP REST API, 실시간 스트리밍을 위한 WebSocket 연결, 이벤트 기반 통합을 위한 웹훅 엔드포인트를 제공합니다.

## 개요

게이트웨이는 PRX 데몬의 일부로 실행되며 다음을 처리합니다:

- **HTTP API** -- 세션 관리, 도구 실행, 설정을 위한 RESTful 엔드포인트
- **WebSocket** -- 실시간 에이전트 상호 작용을 위한 양방향 스트리밍
- **웹훅** -- 통합을 위한 아웃바운드 이벤트 알림
- **미들웨어** -- 인증, 속도 제한, CORS, 요청 로깅

## 아키텍처

```
┌─────────────────────────────────┐
│           Gateway                │
│  ┌──────────┐  ┌─────────────┐  │
│  │ HTTP API │  │  WebSocket  │  │
│  └────┬─────┘  └──────┬──────┘  │
│       │               │         │
│  ┌────┴───────────────┴──────┐  │
│  │       Middleware Stack     │  │
│  └────────────┬──────────────┘  │
│               │                  │
│  ┌────────────┴──────────────┐  │
│  │      Agent Runtime         │  │
│  └───────────────────────────┘  │
└─────────────────────────────────┘
```

## 설정

```toml
[gateway]
bind = "127.0.0.1:3120"
tls_cert = ""
tls_key = ""

[gateway.cors]
allowed_origins = ["*"]
allowed_methods = ["GET", "POST", "PUT", "DELETE"]
```
