# L3: 전략 진화

레이어 3은 가장 높은 영향력과 가장 낮은 빈도의 진화 계층입니다. 에이전트의 전략적 동작 -- 도구 접근 정책, LLM 라우팅 규칙, 거버넌스 파라미터를 수정합니다. 광범위한 영향으로 인해 L3 변경은 항상 명시적 승인을 필요로 합니다.

## 개요

L3 진화가 다루는 영역:

- **도구 정책 튜닝** -- 사용 가능한 도구와 권한 수준 조정
- **라우팅 최적화** -- 성능 데이터를 기반으로 모델 선택 휴리스틱 다듬기
- **거버넌스 파라미터** -- 안전 임계값, 레이트 리밋, 승인 워크플로우 튜닝
- **비용 최적화** -- 품질과 리소스 사용 사이의 균형

## 작동 방식

1. L3이 주/월 단위로 집계 성능 데이터를 수집합니다
2. 패턴을 식별합니다 (예: 더 저렴한 모델이 쿼리의 80%를 적절히 처리)
3. 예상 영향 분석과 함께 전략적 변경을 제안합니다
4. 변경이 사람의 승인을 위해 대기열에 놓입니다
5. 승인된 변경이 자동 롤백 기능과 함께 적용됩니다

## 설정

```toml
[self_evolution.l3]
enabled = false
schedule = "monthly"
require_approval = true
rollback_window_hours = 168  # 7일
max_policy_changes_per_cycle = 3
```
