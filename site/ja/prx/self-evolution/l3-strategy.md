# L3: 戦略進化

レイヤー 3 は、最も影響が大きく最も頻度の低い進化レイヤーです。エージェントの戦略的動作 -- ツールアクセスポリシー、LLM ルーティングルール、ガバナンスパラメーター -- を変更します。広範な影響があるため、L3 の変更には常に明示的な承認が必要です。

## 概要

L3 進化は以下に対処します:

- **ツールポリシーチューニング** -- 利用可能なツールとその権限レベルの調整
- **ルーティング最適化** -- パフォーマンスデータに基づくモデル選択ヒューリスティクスの改良
- **ガバナンスパラメーター** -- 安全性しきい値、レート制限、承認ワークフローの調整
- **コスト最適化** -- 品質とリソース使用のバランス

## 仕組み

1. L3 は数週間/数ヶ月にわたって集約パフォーマンスデータを収集
2. パターンを特定（例: より安価なモデルがクエリの 80% を適切に処理）
3. 予想される影響分析と共に戦略的変更を提案
4. 変更は人間の承認のためにキューに入れられる
5. 承認された変更は自動ロールバック機能付きで適用

## 設定

```toml
[self_evolution.l3]
enabled = false
schedule = "monthly"
require_approval = true
rollback_window_hours = 168  # 7 日間
max_policy_changes_per_cycle = 3
```
